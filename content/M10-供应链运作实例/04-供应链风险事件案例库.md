## 一、为什么要学习风险事件案例？🟢 基础必掌握

真实的供应链风险事件是最好的 SCRM（供应链风险管理）教材：

## 二、案例一：2011 日本地震 — 单一来源的代价 🟢 基础必掌握

### 事件背景

2011年3月11日，日本东北发生 9.0 级地震和海啸，多家核心零部件工厂停产：

- **瑞萨电子**（全球最大汽车 MCU 供应商）：茨城工厂严重受损，停产 3 个月
- **信越化学**：硅晶圆产量下降（半导体原材料）
- **旭硝子**：液晶玻璃基板供应中断

### 供应链影响

| 行业 | 影响 |
|------|------|
| **汽车**（丰田/本田/日产）| 全球减产约 50 万辆，北美工厂因缺关键零件被迫减产 |
| **消费电子**（苹果/三星）| 硬盘供应收紧，PC 厂商库存告急 |
| **半导体**（全球）| 硅晶圆供货量减少约 25%，持续 6 个月 |

### 根本原因分析

```
单一来源（Sole Source）+ 单一地区集中：
  瑞萨控制汽车 MCU 全球 30%+ 市场份额
  该 MCU 的生产几乎全部在日本
  → 一旦日本发生灾害，全球汽车供应链同时断裂
```

### 改进措施（行业普遍执行）

- 推进关键零部件双源甚至三源策略
- 要求供应商将生产分散到多个地理位置
- 对关键物料增加"战略储备"（从 2 周备货提高至 4～8 周）
- 建立供应商层级穿透（Tier 2、Tier 3 透明化，不只看直接供应商）

## 三、案例二：2021 年苏伊士运河堵塞 — 物流单一依赖 🟡 进阶了解

### 事件背景

2021年3月23日，超大型集装箱船"长赐号"（Ever Given）搁浅苏伊士运河，堵塞 6 天。

**规模**：
- 苏伊士运河承担全球约 12% 的贸易量
- 堵塞期间约 400 艘船只被迫绕行或等待
- 估算每天影响约 96 亿美元的货物流通

### 供应链影响

| 行业 | 影响 |
|------|------|
| **消费品/零售** | 欧洲零售商预期收到的季节性商品延误 2～3 周 |
| **工业制造** | JIT 工厂部分零件出现短期断供风险 |
| **能源** | 石油/天然气船运价格短期波动 |

### 风险应对实践

```
短期（立即）：
  ✓ 绕行好望角（增加约 7,000 海里，延误 7～14 天，成本增加约 $200k/船次）
  ✓ 紧急空运替代极关键货物（成本是海运的 6～8 倍）
  ✓ 评估在手库存可以支撑多久

中期（1个月内）：
  ✓ 与下游客户沟通，调整交期承诺
  ✓ 审查并调整在途保险（部分货物因绕行产生额外损耗）

长期（复盘）：
  ✓ 评估物流路线多元化（不完全依赖苏伊士运河）
  ✓ 增加对 JIT 物料的安全库存（降低对精准到货日期的依赖）
```

### 关键教训

> 供应链风险不只来自供应商，**物流节点的集中**同样是风险点。苏伊士运河、巴拿马运河、台湾海峡、马六甲海峡都是全球供应链的咽喉要道。

## 四、案例三：新冠疫情对供应链的冲击（2020）🟢 基础必掌握

### 事件背景

2020年1～3月，中国武汉及多地工厂停产，随后疫情扩散至全球，供应链受到史无前例的冲击。

### 三阶段冲击

```
第一阶段（2020年1～3月）：供应冲击
  中国工厂停产 → 全球企业库存快速消耗 → 部分工厂停线

第二阶段（2020年3～6月）：需求冲击
  全球封锁 → 消费需求骤降（汽车/餐饮/旅游） → 企业砍单
  vs 居家办公/电商需求暴增 → 消费电子需求超预期

第三阶段（2020年下半年～2022年）：不平衡恢复
  供应恢复慢（工厂产能爬坡需要时间）
  需求快速反弹（政府刺激 + 消费需求释放）
  → 结构性缺货（见 M10-01 缺芯危机）
```

### 中国企业的应对

| 企业 | 应对措施 |
|------|---------|
| **华为** | 提前大规模囤积芯片（预判实体清单风险）|
| **比亚迪** | 快速转产口罩（多元化产品，渡过需求低谷）|
| **宁德时代** | 与客户签 LTA 锁定订单，压货反弹期需求 |
| **头部消费电子厂** | 提高安全库存，从 2 周提升至 3～6 个月 |

### 对供应链管理的长远影响

- **韧性 vs 效率的重新平衡**：从"极致效率"向"效率+韧性"转变
- **近岸外包（Nearshoring）**：欧美企业减少对远东供应商依赖，引入墨西哥/东欧供应商
- **供应链可视化投资增加**：企业愿意为"看见"供应链支付溢价
- **BCP 演练从形式变成实战**：企业重新审视 BCP 有效性

## 五、案例四：华为被制裁的供应链应对 🟡 进阶了解

### 背景

2019年5月，华为被列入美国实体清单，随后芯片供应受到严格限制，高通、英特尔、博通等美国芯片企业停止供货。

### 华为的应对策略

```
短期（2019～2020年）：
  ✓ 紧急建立备用芯片库存（据报道囤积约 1 年用量的芯片）
  ✓ 加速海思麒麟自研芯片的量产比例
  ✓ 寻找非美国技术来源的替代供应商

中期（2020～2022年）：
  ✓ 与中芯国际合作（中国最先进晶圆代工厂）
  ✓ 手机业务受创，转向 B2B（企业网络、服务器、云）
  ✓ 出售部分品牌（荣耀）以减少制裁压力

长期（2023年至今）：
  ✓ 2023年 Mate 60 Pro 搭载中芯国际 7nm 工艺麒麟 9000S，突破技术封锁
  ✓ 全面推进软硬件国产化（鸿蒙 OS 替代安卓）
  ✓ 供应链国产替代比例从 30% 提升至 70%+（据估算）
```

### 供应链管理启示

| 启示 | 应用 |
|------|------|
| **单一来源+政治风险 = 极高风险** | 对受出口管制影响的关键物料，提前建立库存缓冲 |
| **自研是终极风险对冲** | 关键技术不能完全依赖外采，核心 IP 必须掌握在自己手中 |
| **供应链国产化是战略** | 不只是商业决策，也是国家战略指引下的必然选择 |

## 六、风险事件应对的通用框架 🟢 基础必掌握

面试时可以用此框架回答"你如何处理突发供应链风险"：

```
Stage 1：感知与评估（0～24小时）
  - 风险源：什么发生了？影响哪些供应商/物料？
  - 影响范围：有多少 PO 受影响？什么时候会断货？
  - 内部通报：立即通知生产/PMC/管理层

Stage 2：应急响应（24～72小时）
  - 紧急盘点：当前安全库存能撑多久？
  - 替代来源：有无备份供应商？能否紧急调货？
  - 生产优先级：资源不足时，哪些订单优先保障？

Stage 3：恢复计划（72小时～30天）
  - 与受影响供应商协商：何时能恢复？部分恢复的节奏？
  - 中长期替代：启动第二供应商认证
  - 客户沟通：与下游客户同步，调整交期承诺

Stage 4：复盘与改进（30天后）
  - 根因分析：风险为什么没有被提前预见？
  - BCP 更新：将本次事件纳入应急预案
  - Scorecard 更新：事件中供应商的表现记入绩效
```

## 七、关键术语速查 🟢

| 术语 | 解释 |
|------|------|
| **SCRM** | Supply Chain Risk Management，供应链风险管理 |
| **BCP** | Business Continuity Plan，业务连续性计划 |
| **Nearshoring** | 近岸外包，将供应源移至地理上更近的国家 |
| **实体清单** | Entity List，美国商务部禁止与之交易的企业名单 |
| **Tier 穿透** | 了解 Tier 2/3 供应商，不只管直接供应商 |
| **战略储备** | Strategic Buffer，对关键物料维持超过正常水平的安全库存 |
| **苏伊士运河** | 连接欧洲与亚洲的全球最重要航运通道 |
