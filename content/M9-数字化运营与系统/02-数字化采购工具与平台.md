## 一、数字化采购的层级体系 🟢 基础必掌握

企业采购数字化工具可以分为三个层级：

```
战略层（Strategy）：
  供应商信息管理（SIM）/ 供应商绩效管理（SRM）/ 支出分析（Spend Analytics）

执行层（Execution）：
  电子寻源（e-Sourcing）/ 合同管理（CLM）/ 电子采购（e-Procurement）/ P2P

运营层（Operations）：
  ERP（SAP/Oracle）/ WMS（仓库管理）/ TMS（运输管理）
```

## 二、主要工具类型详解 🟢 基础必掌握

### 2.1 SRM（供应商关系管理系统）

**SRM（Supplier Relationship Management）**：管理供应商信息、绩效、认证状态的综合平台，相当于采购部门的"供应商 CRM"。

**核心功能**：
- 供应商注册与准入（AVL 管理）
- 供应商 Scorecard 数据录入和计算
- 供应商文件管理（认证证书、协议档案）
- 供应商分级和风险评分

**代表产品**：
- SAP Ariba（全球最大，大型企业主流）
- Jaggaer（中型企业常用）
- 阿里云采购（国内企业）
- 用友采购云（国内）

---

### 2.2 e-Sourcing（电子寻源平台）

**e-Sourcing**：在线发起 RFI/RFQ/RFP，供应商线上报价和评标的平台。

**传统方式** vs **e-Sourcing**：

| 对比 | 传统邮件 RFQ | e-Sourcing 平台 |
|------|------------|----------------|
| 报价收集 | 邮件分散，手工整理 Excel | 统一平台在线填写，自动汇总 |
| 供应商管理 | 手工维护供应商列表 | 平台自动匹配、分级 |
| 报价保密性 | 邮件可转发，难以保密 | 封闭系统，开标前自动锁定 |
| 审计追踪 | 邮件记录，难以查找 | 全程留痕，自动生成审计报告 |
| 在线反向拍卖 | 不支持 | 支持（e-Auction，实时竞价）|

**e-Auction（反向拍卖）**：
```
采购方发布需求 → 多家供应商实时在线出价 → 价格不断下降 → 设定时间结束后最低价胜出
```
适用于：标准化物料、竞争充分的品类（如通用电子元件、大宗原材料）

---

### 2.3 CLM（合同生命周期管理）

**CLM（Contract Lifecycle Management）**：管理合同从起草、谈判、签署到执行监控、续签/终止的全生命周期。

**解决的痛点**：
- 合同分散在各人员邮箱 / 本地文件夹，难以统一管理
- 合同到期提醒缺失，导致自动续约或错失谈判窗口
- 合同版本混乱，无法追踪修改历史
- 法务审核流程缺乏标准化

**代表产品**：SAP Ariba Contracts、Icertis（全球领先）、阿里云合同管理

---

### 2.4 支出分析（Spend Analytics）

**Spend Analytics**：对企业所有采购支出数据进行清洗、分类、可视化分析，找出降本机会。

**典型分析维度**：

```
支出分析仪表盘（Spend Dashboard）：
├── 按品类（Category）分布：芯片/PCB/结构件/MRO
├── 按供应商分布：Top 10 供应商占总支出的比例
├── 按地区分布：国内 vs 进口
├── 按时间趋势：月度/季度支出波动
└── 尾部供应商（Long Tail）：识别小额分散采购，合并节省成本
```

**关键指标**：
- **尾部采购占比**：金额 < 总支出 20% 的供应商数量，通常占总供应商数的 80%，是整合降本的主要机会
- **单一来源比例**：只有 1 家供应商的品类占比，反映供应链风险集中度

---

### 2.5 WMS（仓库管理系统）

**WMS（Warehouse Management System）**：管理仓库入库、出库、上架、盘点、库位的专业系统。

| 功能 | 说明 |
|------|------|
| 入库管理 | GR 扫码确认、货位分配 |
| 库位管理 | 货物精确到"区-排-列-层" |
| 出库管理 | 拣货路径优化（减少行走距离）|
| 盘点管理 | 移动设备扫码盘点，自动对账 |
| 先进先出（FIFO）| 系统强制按照入库时间顺序出库 |

**代表产品**：SAP EWM（扩展仓库管理）、曼荼罗 WMS、库卡 WMS、菜鸟仓储

---

### 2.6 TMS（运输管理系统）

**TMS（Transportation Management System）**：管理货物运输计划、承运商选择、运费结算、全程可视化的系统。

**采购人关注点**：
- 入厂物流（Inbound Logistics）的 TMS 可以追踪在途库存的实时位置
- 供应商发货后在 TMS 中录入运单号 → 采购方实时查看货物位置
- ASN（提前发货通知）通常集成在 TMS 中

## 四、电子数据交换（EDI）🟡 进阶了解

**EDI（Electronic Data Interchange，电子数据交换）**：企业间通过标准化格式自动传输业务文件（PO、发票、ASN）的技术，无需人工操作。

**EDI 的工作流程**：
```
买方 ERP 生成 PO
    ↓ EDI 自动发送（ANSI X12 / EDIFACT 格式）
供应商 ERP 接收 PO → 自动确认
    ↓ EDI 自动回传
买方 ERP 收到 PO 确认
    ↓ 发货后
供应商 ERP 发送 ASN → EDI → 买方 ERP 自动创建预期收货
```

**适用场景**：
- 大批量、高频次的标准化订单（如汽车主机厂 → 零件供应商）
- 年交易额超过 500 万元的供应商
- 需要全程自动化、减少人工操作的场景

## 五、供应链可视化与风险监控工具 🔴 暂缓

**供应链可视化（Supply Chain Visibility）**：实时追踪供应链上每个节点（供应商产能、运输位置、港口状态）的数字化能力。

代表产品：
- Resilinc（供应链风险监控，实时追踪工厂停工/自然灾害事件）
- project44（运输可视化）
- Shippeo（欧洲运输追踪）
