# 03 订单执行与交期管理

> **📌 知识层级说明**
> 🟢 **基础必掌握** — 入职前、面试前必须理解
> 🟡 **进阶了解** — 有基础后扩展，理解逻辑即可
> 🔴 **暂缓** — 工作后遇到实际场景再深入

---

## 一、从 PR 到 PO 的执行流程 🟢 基础必掌握

```
PMC/需求部门发起 PR（采购申请）
        ↓ 审批通过
Buyer 审核 PR，匹配 AVL 合格供应商
        ↓
在 ERP 系统中创建 PO（采购订单）
        ↓
PO 发送给供应商（邮件 / ERP 系统 / EDI）
        ↓
供应商确认 PO（Acknowledge）
        ↓
进入跟单阶段（Expediting）
        ↓
供应商发货（+提前发 ASN 预报）
        ↓
货物到厂，完成 GR（收货）
```

---

## 二、PO 的关键字段 🟢 基础必掌握

一张完整的 PO 必须包含：

| 字段         | 说明               |
| ---------- | ---------------- |
| **PO 编号**  | 唯一识别号，贯穿整个采购生命周期 |
| **供应商信息**  | 名称、地址、税号、银行账户    |
| **物料信息**   | 物料编号、规格描述、图号     |
| **数量**     | 采购数量 + 单位        |
| **单价**     | 含税/不含税，币种        |
| **交货日期**   | 要求到厂日期（非发货日期）    |
| **交货地点**   | 具体收货仓库地址         |
| **付款条件**   | Net 60 / 预付比例等   |
| **质量要求**   | 参考规格书编号 / 检验标准   |
| **PO 有效期** | 逾期未执行的处理方式       |

## 三、供应商确认 PO（Acknowledge）🟢 基础必掌握

PO 发出后，必须要求供应商在规定时间内（通常 24～48 小时）正式确认：

**供应商确认的内容**：
- 数量是否接受（如产能不足可能拒绝全量）
- 交货日期是否可以达到（可能给出比要求晚的日期）
- 价格是否无异议

**三种回应情况的处理**：

| 供应商回应 | 采购处理方式 |
|---------|------------|
| **全部接受（Accept）** | PO 正式生效，进入跟单阶段 |
| **部分接受（Partial）** | 与 PMC 确认缺口影响，决定是否找备份供应商补足 |
| **拒绝 / 无回应** | 催促确认，超时未回则上报，评估启动备份供应商 |

> 💡 **实战经验**：供应商确认看似简单，但"收到 PO 不回复"在中小供应商中极为常见。应建立明确的 SLA（服务水平协议）：要求24小时内必须书面确认，否则视为违约。

## 四、Expediting（跟单催货）🟢 基础必掌握

**Expediting**：在 PO 生效到货物到厂期间，主动跟踪供应商的生产和交货进度，提前发现风险并干预。

### 跟单节奏参考

| 交货周期 | 建议跟单频率 |
|---------|-----------|
| Lead Time < 2周 | 临近交期前3天确认一次 |
| Lead Time 2～8周 | 每周跟进一次 |
| Lead Time > 8周 | 每2周跟进一次（含里程碑节点：原材料采购→生产启动→完工→发货）|
| 关键/紧急物料 | 每日跟进，要求供应商每日更新进度 |

### 跟单核查清单

- [ ] 原材料是否已备齐？
- [ ] 生产是否已按计划启动？
- [ ] 预计完工时间是否与交期一致？
- [ ] 质检是否通过？
- [ ] 包装和出货准备是否就绪？
- [ ] 运输安排是否已确认（承运商/物流单号）？

## 五、ASN（提前发货通知）🟡 进阶了解

**ASN（Advanced Shipment Notification，提前发货通知）**：供应商在发货前（通常提前 24～48 小时）向采购方发出的通知，告知即将到货的详细信息。

**ASN 包含内容**：
- PO 编号
- 物料编号、发货数量
- 发货日期、预计到货日期
- 物流公司、运单号
- 包装方式（几箱、几托盘）

**ASN 的价值**：
- 仓库可提前安排卸货资源和货位
- 质检可提前准备检验计划
- 如果 ASN 数量与 PO 不符，可提前处理分歧，避免到货时混乱

## 六、交期延误的处理流程 🟢 基础必掌握

延误是采购日常最高频的挑战，处理流程如下：

```
Step 1：收到延误信号（供应商主动告知 or 跟单发现）
        ↓
Step 2：确认延误程度（延误多少天？影响哪些 PO？影响哪些生产计划？）
        ↓
Step 3：内部通报（立即通知 PMC、生产计划，评估影响）
        ↓
Step 4：与供应商沟通原因和补救方案
          - 是否可以分批发货（先发能交的部分）？
          - 是否可以优先排我们的订单（生产优先级）？
          - 是否需要加班或增加资源？
        ↓
Step 5：若无法及时补救 → 启动备份供应商或现货采购
        ↓
Step 6：延误严重时 → 按合同追究违约责任（违约金通知）
        ↓
Step 7：事后复盘 → 记录在供应商 Scorecard，OTIF 扣分
```

## 七、关键术语速查 🟢

| 术语 | 全称/解释 |
|------|---------|
| **PO** | Purchase Order，采购订单 |
| **PR** | Purchase Requisition，采购申请 |
| **Acknowledge** | 供应商正式回复确认 PO |
| **Expediting** | 跟单催货行为 |
| **ASN** | Advanced Shipment Notification，提前发货通知 |
| **Lead Time** | 从下单到收货的周期（天/周）|
| **OTD** | On-Time Delivery，准时交货率（只考虑"准时"，不考虑"完整"，区别于 OTIF）|
| **Split Delivery** | 分批交货，大 PO 分多次到货 |
| **Short Shipment** | 短交，实际交货量少于 PO 数量 |
